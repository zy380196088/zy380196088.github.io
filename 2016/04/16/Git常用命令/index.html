Git常用命令

发表于 2016-04-16

用户信息

git config --global user.name “youname”
git config --global user.email “123456@example.com”
## Git基础 ### 查看配置信息 
git config --list
git config -l

初始化新仓库

git init
初始化后，在当前目录下会出现一个名为 .git 的目录，所有 Git 需要的数据和资源都存放在这个目录中。 ### 跟踪文件 
git add

跟踪全部文件

giat add .

提脚更新

git commit -m "跟新描述"

添加原程仓库

git remote add [shortname] [url]
例如: 
git remote add origin git@github.com:yourname/yourRepo.git

上传推送到github

git push [remote-name] [branch-name]
git push origin master

如果要把本地的 master 分支推送到origin服务器上(再次说明下，克隆操作会自动使用默认的master 和origin 名字)

从远程仓库克隆

git clone git://github.com/youname/project.git
git clone git@github.com:youname/project.git
git clone https://github.com/youname/project.git

从原程仓库抓取数据

git fetch [remote-name]

删除原程仓库

git remote rm otigin

检查当前文件状态

git status

创建.gitginore文件

touch .gitignore

查看修改之后 没有暂存起来的内容

git diff

移除文件

git rm 文件名.后缀

重命名文件

git mv oldname.后缀 newname.后缀

查看提交历史

git log

取消暂存

git reset 文件名

Git分支

创建名为name的分支

git branch name

切换到name分支（默认master分支）

git checkout name

上面两条命令可缩写为：

git checkout -b name

删除name分支（如果该分支还未被合并则会提示错误，因为这样会丢失数据）

git branch -d name

强制删除：

git branch -D name

合并分支（先切换到master分支）

git checkout master
git merge name

查看当前所有分支

git branch

查看各个分支最后一个提交对象的信息

git branch -v

查看已经与当前分支合并的分支（已经合并的查出来后可以删掉）

git branch --merge

查看未与当前分支合并的分支

git branch --no-merged
